3. Missions réalisées

3.1 Développement d'une application d'analyse de concours Instagram

La mission principale de ce stage consistait à développer une application permettant d'analyser les participants aux jeux-concours Instagram d'Olevia et de sélectionner automatiquement un gagnant parmi les personnes respectant toutes les règles.

Contexte et objectifs :

Les jeux-concours Instagram (ex: "Likez, commentez et tagguez 3 amis pour gagner") sont un excellent moyen d'augmenter l'engagement et la visibilité d'une marque. Cependant, vérifier manuellement que chaque participant a bien respecté toutes les règles (like, follow, mentions) et sélectionner un gagnant parmi des centaines de commentaires est chronophage et source d'erreurs.

L'objectif était donc de créer une application capable de :

  • Analyser le code source HTML des pages Instagram (likes, abonnés, commentaires)
  • Extraire et stocker les données dans une base de données SQL
  • Vérifier automatiquement que les participants respectent les 3 critères (like, follow, tags)
  • Sélectionner aléatoirement un gagnant parmi les participants éligibles

Technologies utilisées :

  • Python 3 - Langage principal pour l'analyse
  • BeautifulSoup4 - Parsing et analyse du HTML Instagram
  • Playwright - Bibliothèque pour navigateur (dépendance)
  • MySQL - Base de données pour stocker les participants
  • mysql-connector-python - Connexion Python ↔ MySQL

Architecture de l'application :

AnalyseConcour/ ├── main.py # Sélection du gagnant final ├── Analyse_Like/ │ ├── analyse_like.py # Extraction des likes │ └── page.html # HTML de la page des likes ├── Analyse_Subscribe/ │ ├── analyse_Subscribe.py # Extraction des abonnés │ └── page.html # HTML de la page des abonnés ├── Analyse_Mention/ │ ├── analyse_mention.py # Extraction des commentaires │ └── page.html # HTML de la page des commentaires └── requirements.txt

Processus de fonctionnement :

Étape 1 - Récupération du HTML :

Le code source HTML de chaque page Instagram (liste des likes, liste des abonnés, commentaires de la publication) est récupéré manuellement et sauvegardé dans un fichier page.html dans chaque dossier d'analyse. Cette approche permet de contourner les limitations strictes de l'API Instagram et d'éviter les détections anti-bot.

Étape 2 - Analyse des Likes (analyse_like.py) :

  • Parse le HTML avec BeautifulSoup pour extraire tous les pseudos ayant liké le post
  • Recherche les balises <span class="_ap3a"> contenant les noms d'utilisateur
  • Injecte chaque pseudo dans la table SQL likepost
  • Gère les doublons avec ON DUPLICATE KEY UPDATE

Étape 3 - Analyse des Abonnés (analyse_Subscribe.py) :

  • Parse le HTML de la liste des abonnés du compte Olevia
  • Identifie les vrais abonnés en vérifiant la présence du bouton "Supprimer" dans chaque bloc utilisateur
  • Extrait les pseudos avec la classe CSS _ap3a
  • Injecte dans la table SQL subscribe

Étape 4 - Analyse des Mentions (analyse_mention.py) :

  • Parse les commentaires de la publication du concours
  • Pour chaque commentaire, extrait l'auteur et toutes les mentions (@username)
  • Applique des filtres de validation :
    • Minimum 3 mentions différentes requises
    • Exclut les mentions du compte hôte (@tacoblizt)
    • Exclut l'auto-mention (l'auteur ne peut pas se taguer lui-même)
  • Injecte dans la table SQL mention : auteur + 3 premières mentions

Étape 5 - Sélection du gagnant (main.py) :

Le script principal effectue une requête SQL qui croise les 3 tables pour trouver les participants éligibles :

SELECT pseudo, message_1, message_2, message_3 FROM mention WHERE pseudo IN (SELECT pseudo FROM subscribe) AND pseudo IN (SELECT pseudo FROM likepost) AND pseudo IN (SELECT pseudo FROM mention) ORDER BY RAND() LIMIT 1;

Cette requête sélectionne aléatoirement 1 personne qui apparaît simultanément dans les 3 tables, garantissant qu'elle a bien liké, suivi le compte, et commenté avec au moins 3 mentions valides.

Structure de la base de données :

Table Colonnes principales Rôle
likepost pseudo, date_collecte Liste des personnes ayant liké la publication
subscribe pseudo, date_collecte Liste des abonnés au compte Olevia
mention pseudo, message_1, message_2, message_3, date_collecte Commentaires valides avec ≥3 mentions

Défis techniques rencontrés :

  • Structure HTML complexe d'Instagram : Instagram utilise des classes CSS générées automatiquement (ex: _ap3a, x9f619) qui peuvent changer sans préavis. Solution : ciblage précis des balises et tests réguliers de la robustesse du parsing.
  • Détection des vrais abonnés : Différencier les abonnés actifs des comptes suspendus. Solution : vérification de la présence du bouton "Supprimer" supprimable dans chaque bloc.
  • Validation des mentions : Filtrer les commentaires invalides (mentions insuffisantes, auto-tags, mentions du compte hôte). Solution : logique de filtrage multi-critères avec exclusions.
  • Gestion des doublons SQL : Un utilisateur peut apparaître plusieurs fois dans les extractions. Solution : utilisation de INSERT IGNORE et ON DUPLICATE KEY UPDATE.

5. Difficultés rencontrées et solutions apportées

5.1 Compréhension de la structure HTML Instagram

Difficulté : Instagram utilise des classes CSS générées automatiquement et une structure HTML complexe qui rend difficile l'extraction des données. Il fallait identifier les bonnes balises et classes pour extraire les pseudos, les likes, les abonnés et les commentaires de manière fiable.

Solution : Analyse approfondie du HTML avec les outils de développement du navigateur, tests répétés pour identifier les sélecteurs CSS les plus stables (_ap3a, x9f619), et développement d'une logique de parsing robuste avec BeautifulSoup capable de gérer les variations.

5.2 Apprentissage de Java/Spring Boot

Difficulté : Transition de PHP (procédural/orienté objet simple) vers Java avec Spring Boot (framework complexe, annotations, concepts avancés).

Solution : Formation progressive avec tutoriels (Spring.io, Baeldung), création de petits projets d'apprentissage, lecture de la documentation officielle, et assistance de la communauté (Stack Overflow).
